# Devin Memory System

## Overview

Devin (by Cognition) uses a multi-layered memory system: a Knowledge base for organizational context, Playbooks for reusable task templates, session state with rewind capability, and auto-generated insights. Devin operates as a cloud-hosted autonomous agent, so memory is managed server-side.

## Storage Locations

| Component | Location | Scope |
|-----------|----------|-------|
| Knowledge | Devin cloud (web app) | Organization-wide |
| Playbooks | Devin cloud (web app Library) | Organization or personal |
| Session state | Devin cloud (per-session) | Per-session, rewindable |
| Auto-insights | Devin cloud | Organization-wide |

All storage is cloud-hosted. There are no local files or directories.

## Memory Types

### Knowledge Base

A persistent collection of tips, instructions, and organizational context. Users add entries over time; Devin automatically recalls relevant knowledge during sessions.

Typical content:
- Best practices and style guides
- Project-specific instructions
- Infrastructure and deployment details
- Team conventions and preferences
- Known gotchas and workarounds

Knowledge is globally available to all Devin sessions within the organization. The agent retrieves relevant entries based on task context.

### Playbooks

Reusable task templates -- like a custom system prompt for a repeated task. Playbooks define step-by-step workflows for common operations.

- Created manually by users or auto-generated by Devin (in Advanced mode)
- Shareable across the organization
- Community playbook library available
- Can be iteratively refined (Devin can be used to improve playbooks)

Examples: "Deploy to staging", "Create a new API endpoint following our patterns", "Run the full QA suite and fix failures."

### Session State (with Rewind)

Within a session, Devin maintains full context: file system state, conversation history, and execution state. The rewind feature allows restoring a session to any previous point in time, resetting both memory and file system.

Use cases for rewind:
- Undoing wrong decisions
- Exploring alternative approaches
- Iterating on playbook/prompt refinements

### Auto-Generated Insights

Devin generates insights from successful and failed sessions. In Advanced mode, Devin can inspect other sessions to create and improve playbooks automatically, proposing batch sessions for similar tasks.

## Persistence Scope

- **Knowledge**: Persists indefinitely across all sessions. Manually managed.
- **Playbooks**: Persist indefinitely. Version-controllable through the web interface.
- **Session state**: Persists for the session lifetime. Can be rewound but not carried to new sessions.
- **Cross-session learning**: Devin's architecture maintains context across extended sessions but does not automatically carry specific memories between separate sessions (knowledge base is the mechanism for that).

## User Control

- Users add, edit, and remove Knowledge entries via the web app.
- Playbooks can be created, edited, shared, and deleted from the Library.
- Session rewind gives granular control over in-session state.
- Advanced mode enables Devin to self-improve playbooks.
- No local file access to memory -- everything is through the web interface.

## Implementation Details

- Devin combines LLMs with specialized components for planning, execution, and verification.
- Knowledge retrieval appears to use relevance-based selection (not full injection of all knowledge).
- The system learns from failures within a session and adapts its approach.
- Playbooks function as structured prompts that constrain and guide agent behavior.
- Advanced mode unlocks meta-capabilities: inspecting sessions, auto-creating playbooks, batch operations.
- Cloud-hosted architecture means memory is tied to the Cognition platform.

## Strengths

- **Rewind capability**: Unique among agents. Allows exploration of alternative approaches without losing state.
- **Knowledge base scales with the organization**: Team members collectively build institutional knowledge.
- **Playbooks for repeatability**: Codified workflows reduce variability in common tasks.
- **Auto-improvement**: Advanced mode lets Devin refine its own playbooks based on outcomes.
- **Community library**: Shared playbooks provide starting points for common patterns.

## Weaknesses

- **Cloud-only, vendor-locked**: All memory is on Cognition's servers. No local access, no export, no portability.
- **No file-system memory artifacts**: Unlike CLAUDE.md or .cursorrules, there's nothing in the repo to version-control.
- **Opaque retrieval**: Users can't see exactly which knowledge entries influenced a decision.
- **No cross-session auto-memory**: Knowledge must be manually added. Devin doesn't auto-persist session learnings to the knowledge base (except via Advanced mode's playbook improvement).
- **Session state is ephemeral**: Once a session ends, its context is gone (beyond what's in the knowledge base).
- **Cost**: Devin's cloud-hosted model means memory features are tied to subscription pricing.
